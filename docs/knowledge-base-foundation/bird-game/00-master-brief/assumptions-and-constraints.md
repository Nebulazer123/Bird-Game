# Assumptions and Constraints

## Assumptions
- The current repo is useful source material, not something sacred.
- Aggressive reshape or near-restart of major gameplay systems is allowed if it leads to a better game.
- The best near-term target is a polished, unforgettable vertical slice rather than immediate expansion into a larger full game.
- Flight feel should be treated as the first major pillar to improve before adding more feature breadth.
- Combat is not automatically core; it must justify itself against the bird-flight fantasy.
- UI/UX work for this project means readability and feel communication, not just menu prettiness.

## Constraints
- The game already exists in Three.js + Vite and includes real assets, automated Playwright tests, and a working gameplay loop.
- The current gameplay logic is concentrated mostly in `src/game.js`, which increases the risk of further feature growth without structural change.
- The planning outputs need to be strong enough that later specialist chats can continue without needing the original conversation.
- The user prefers serious planning first, not rushed implementation.

## Available Subscriptions and Tools
- OpenAI and Google APIs are allowed.
- ChatGPT Pro, Gemini Pro, and GitHub Copilot Pro are already available.
- Installed skills include `research-kb`, `ui-ux-pro-max`, `architecture-planner`, `tooling-and-third-party-scout`, `implementation-planner`, `synthesis-analyst`, `develop-web-game`, and supporting helpers.

## Workflow Preferences
- Prefer exhaustive planning over rushed output.
- Avoid code-heavy manual work for the user.
- Minimize repeated prompting between specialist handoffs.
- Favor free-first or local-first options when practical.
- Treat feel, fantasy, and direction quality as more important than preserving existing systems.
