# Success Criteria

## Success Looks Like
- The project has a clear answer to what fantasy the game should deliver.
- The planning outputs make it obvious whether combat belongs in the strongest version of the game.
- The team has a strong recommendation on whether to build on the current structure or substantially replace it.
- Flight feel, camera clarity, speed readability, danger readability, and feedback clarity are treated as primary design concerns.
- The specialist outputs can be synthesized into a build-ready direction for a genuinely special vertical slice.

## Failure Looks Like
- The plan assumes the current loop is correct without challenging it.
- Combat, rings, upgrades, and stage progression are all kept by default just because they already exist.
- UI/UX is reduced to cosmetic HUD polish rather than readability and control communication.
- The resulting handoffs are too vague for downstream specialist chats to act on confidently.
- The project leaves this phase still unsure what makes the game special.

## Quality Bar
- Clear, decision-oriented planning rather than generic brainstorming.
- Honest product judgment, including willingness to recommend removing or replacing existing systems.
- Specialist prompts that are specific, grounded in the repo, and written like capable teammate handoffs.
- Outputs should support a premium-feeling action game direction, not merely a functional prototype cleanup.
